# 機制系統設計文件

## 1. 陷阱系統

### 基本規則
- 玩家在地圖格子上秘密放置陷阱
- 陷阱對設置者本人無效
- 其他玩家踩中後觸發效果並支付金幣給設置者
- 每位玩家同時最多放置2個陷阱
- 放置陷阱的格子對其他玩家不可見
- 陷阱觸發一次後消失
- 持有探測礦石棒（$10）可偵測前方格子是否有陷阱

### 陷阱完整列表
（詳見道具系統文件）

### 陷阱等級升降機制（命運卡觸發）
擲D20判定成功與否，前端顯示擲出X到Y視為成功。
- 升級：陷阱效果提升至上一層級
- 降級：陷阱效果降至下一層級
- 失敗：依卡片設計有不同後果

## 3. 骰子系統

### 可用骰子
D4、D6、D8、D10、D12、D20

### 前端顯示規則
所有判定機制一律以「擲出X到Y視為成功」的方式顯示給玩家，不顯示百分比。

### 後端邏輯
後端使用隨機數字產生（Math.random()），骰子面數決定隨機範圍。
系統自動計算所有商店內容、寶箱品質、隨機事件，玩家不需手動計算。

---

## 4. 玩家介面系統

### 玩家選擇
- 4個玩家按鈕（Player 1–4）
- 當前行動玩家高亮顯示
- 每位玩家可自訂名稱與顏色／符號
- 顯示：金幣數量、道具欄、地圖位置

### 骰子面板
- D4、D6、D8、D10、D12、D20各自獨立按鈕
- 可同時選多顆骰子（例如2D6）
- 顯示每顆結果與總和
- 保留最近幾次擲骰紀錄
- 骰子面板固定在畫面下方或側邊欄

## 5. 勝利點數系統

| 條件 | 勝利點數 |
|------|--------|
| 最終Boss最後一擊 | +3點 |
| 參與擊殺最終Boss | +1點 |
| 最先抵達終點 | +3點 |
| 擊殺史詩級Boss | +2點 |
| 擊殺中級Boss | +1點 |
| 成功觸發陷阱讓對手中招（每次）| +1點 |
| 持有最多金幣者（結算時）| +2點 |
| 持有最多道具者（結算時）| +1點 |
| 抽到終極試煉命運卡並勝利 | +2點 |

### 遊戲結束條件
1. 擊敗最終Boss，最後一擊的玩家勝利
2. 其他玩家全部死亡，活到最後的玩家勝利
3. 第一個走到最終Boss格的玩家，對最終Boss傷害全面提升（+25%）

累積最多勝利點數的玩家最終獲勝。

## 6. 無PVP機制
玩家之間沒有直接戰鬥。
所有玩家互動透過以下間接方式進行：
- 陷阱系統
- 詛咒之石
- 標籤刻印石
- 機會／命運卡片負面效果
- 時間操控道具
- 死亡商店競標
- 最終Boss最後一擊競爭
